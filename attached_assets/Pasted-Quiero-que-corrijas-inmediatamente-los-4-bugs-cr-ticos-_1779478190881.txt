Qué hacer:

* Decidir una sola fuente de verdad:

BUG CRÍTICO #2 — Caja muestra ventas pendientes como cobradas

ventas.metodo_pago tiene DEFAULT 'efectivo', entonces CajaController considera cobradas ventas que nunca fueron pagadas.

Qué hacer:

* eliminar DEFAULT 'efectivo' de ventas.metodo_pago
* permitir null real
* revisar TODO el sistema buscando lógica que use:
  !empty($venta->metodo_pago)

  estado === 'pagado'
  o existencia real de pagos relacionados

BUG CRÍTICO #3 — saldo_favor_usado duplica ingresos
Problema:
CobranzaService::aplicarSaldoFavor() crea Movimiento tipo='ingreso' cuando se usa un saldo a favor. Ese dinero ya entró antes al sistema. El balance queda inflado.

Qué hacer:

* eliminar completamente ese ingreso duplicado
* aplicar saldo NO debe aumentar caja
* si necesitas trazabilidad:
  crear movimiento neutro/contable
  o simplemente registrar observación sin afectar balance
* revisar balances históricos para confirmar que no se duplique nada
